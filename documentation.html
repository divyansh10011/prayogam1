Technical Documentation

Deep dive into the architecture and algorithms powering SAFENET.

Technical Architecture

SAFENET is divided into three primary decoupled layers:

  • Edge Layer (ztas-ai-widget.js): Handles DOM integrity, real-time input monitoring, and state synchronization across the ecosystem.
  • Logic Layer (safe.js): Orchestrates the 30+ threat categories, URL forensic analysis, and character-level sweep.
  • ML Enrichment Layer (safe.py): A heavy-duty FastAPI backend running PyTorch-based Vision Transformers and Scikit-Learn classifiers for deep media forensics.

Threat Detection Modules

Linguistic Threat Detection Engine (ZTAS-NLP)

  • Homograph Attack Detection: Scans for lookalike Unicode characters.
  • Zero-Width Analysis: Detects invisible steering characters used to bypass URL filters.
  • Urgency & BEC Heuristics: Identifies linguistic markers of Business Email Compromise.

Visual Deepfake Detection (EDD_PIXEL_v1.3)

A 7-layer ensemble forensic engine combining Vision Transformers (ViT), Error Level Analysis (ELA), Fast Fourier Transform (FFT) for algorithmic gridding, and Noise Fingerprinting (PRNU).

Triple-Lock Security Pipeline

Every scan passes through three distinct Locks:

  1. Technical Lock: Whitelist checks, HTTPS enforcement, and Safe Browsing API.
  2. Forensic Lock: Heuristic, Spectral, and Entropy analysis (core ZTAS engine).
  3. Logic Lock: LLM-powered synthesis of findings to differentiate False Positives and Stealth Attacks.

Threat scores are exponentially compounded based on cross-vector correlation to flag Advanced Persistent Threats (APTs).
